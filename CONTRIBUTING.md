# Руководство по внесению изменений

Спасибо за интерес к проекту! Вот несколько простых правил для контрибьюторов.

## Как внести изменения

1. **Форкните репозиторий** и создайте ветку для вашей функции/исправления:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Внесите изменения** в код, следуя стилю проекта:
   - Используйте `black` для форматирования кода
   - Проверьте код линтером `ruff`
   - Добавьте тесты для новой функциональности (если применимо)

3. **Убедитесь, что тесты проходят**:
   ```bash
   pytest
   ```

4. **Создайте Pull Request** с описанием изменений.

## Стиль кода

- Используйте аннотации типов где возможно
- Добавляйте docstrings для функций и классов
- Следуйте PEP 8 (автоматически через `black`)

## Сообщения коммитов

Используйте понятные сообщения коммитов на русском или английском языке.
